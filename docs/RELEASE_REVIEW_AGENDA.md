# Release Review Agenda

## Purpose

This agenda is executed after each completed release before planning the next release.
The goal is not only to review code changes, but also to review assumptions,
workflows, agents, documentation, context management and project direction.

## 1. Is our overall approach still correct?

**Questions:**

- Would we start the project the same way today?
- Are we still moving in the right direction?
- Which assumptions have changed?

## 2. What are others doing better?

**Review:**

- New models
- New agent systems
- New workflows
- New tooling

**Questions:**

- What can we learn?
- Which ideas became standard?
- Which ideas became obsolete?

## 3. Which project assumptions are outdated?

**Review:**

- Architecture assumptions
- Workflow assumptions
- Agent assumptions
- Documentation assumptions

## 4. Where do costs currently occur?

**Review:**

- API costs
- Human effort
- Context size
- Planning effort
- Review effort

## 5. Which information do agents really need?

**Review:**

- Planner
- Solver
- Reviewer
- Watchdog

**Questions:**

- What information is required?
- What information is noise?

## 6. Which information should be forgotten?

**Review:**

- Current context
- Temporary observations
- Old discussions
- Expired assumptions

## 7. Is our decomposition level still correct?

**Questions:**

- Are issues too large?
- Are issues too small?
- Should more decomposition happen internally?
- Should more decomposition be visible?

## 8. Which architectural decisions need review?

**Review:**

- Agents
- Skills
- Workflows
- Labels
- Documentation

## 9. Which new ideas deserve an experiment?

**Questions:**

- Which hypotheses should be tested?
- Which ideas deserve a prototype?

## 10. Which findings should become long-term project knowledge?

**Review:**

- Stable principles
- Long-term decisions
- Reusable patterns

## 11. Does this agenda itself need changes?

**Questions:**

- Which sections provided value?
- Which sections should be removed?
- Which sections are missing?

The agenda itself is continuously improved.
